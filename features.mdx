## Custom tokens

Create branded loyalty tokens that your customers can hold in their wallets.

### Token configuration

Full control over your token economics:

```typescript
{
  name: "Coffee Shop Loyalty Program",
  tokenSymbol: "COFFEE",
  totalSupply: "10000",
  tokensPerClaim: "10",
  decimals: "9",
  blockchain: "solana" // or "ethereum"
}
```

### Built-in features

- **Supply caps** - Set maximum token supply to create scarcity
- **Custom metadata** - Add images, descriptions, and external links
- **Multi-chain support** - Deploy on Solana or Ethereum
- **Standard compliance** - ERC-20 compatible tokens

<Note>
  Tokens are created as real blockchain assets that customers can hold, trade, or redeem according to your program rules.
</Note>

## Fraud prevention

Built-in security features protect your campaigns from abuse.

### Protection mechanisms

**One-time use validation**

Each QR code can only be claimed once:

```typescript
// Claim validation schema
{
  tokenId: string,        // Campaign identifier
  uniqueId: string,       // Unique QR code ID
  walletAddress: string,  // Claiming wallet
  wallet: "solana"        // Wallet type
}
```

**Wallet claim limits**

Set maximum claims per wallet address:

```typescript
{
  maxClaimsPerWallet: "5" // Limit each wallet to 5 claims
}
```

**Campaign expiration**

Automatically disable claims after a set date:

```typescript
{
  expirationDate: "2026-12-31" // Optional campaign end date
}
```

<Warning>
  Fraud prevention is automatic and cannot be bypassed. All claims are validated on-chain before tokens are transferred.
</Warning>

## Multi-wallet support

Customers can claim tokens using their preferred wallet provider.

### Supported wallets

- **Phantom** - Popular Solana wallet
- **WalletConnect** - Universal wallet connection protocol
- **Other providers** - Extensible wallet integration

### Claiming flow

When customers scan a QR code:

<Steps>
  <Step title="Scan QR code">
    Customer scans the QR code with their mobile device
  </Step>
  <Step title="Connect wallet">
    Choose their preferred wallet provider (Phantom, WalletConnect, etc.)
  </Step>
  <Step title="Validate claim">
    System validates the QR code and wallet eligibility
  </Step>
  <Step title="Receive tokens">
    Tokens are instantly transferred to their wallet
  </Step>
</Steps>

<Info>
  The claim page is located at `/claim` and handles all wallet connections and token transfers automatically.
</Info>
